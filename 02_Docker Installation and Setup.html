2. Core Docker Concepts

2.1 What is Docker?

Docker is a containerization platform used to build, ship, and run applications in isolated environments called containers. A container packages application code, runtime, libraries, tools, and configuration required by the application.

Simple explanation: Docker solves the common problem: “It works on my machine, but not on another machine.” With Docker, the application runs inside the same packaged environment everywhere.

2.2 Container vs Image

Term Meaning Example
Image A read-only template used to create containers. nginx:latest, ubuntu:24.04
Container A running instance of an image. A running Nginx web server created from the nginx image.
Registry A place where images are stored and shared. Docker Hub, private registry, GitHub Container Registry.
Dockerfile A text file containing instructions to build a Docker image. FROM nginx, COPY index.html /usr/share/nginx/html/

2.3 Image and Container Relationship

2.4 Container vs Virtual Machine

Point Container Virtual Machine
Operating System Shares the host OS kernel. Runs a full guest OS.
Startup Time Usually seconds. Usually minutes.
Size Usually MBs to a few GBs. Usually several GBs.
Isolation Process-level isolation using namespaces and cgroups. Hardware-level isolation using a hypervisor.
Best Use Application packaging and deployment. Running different full operating systems.

3. Docker Architecture

Docker uses a client-server architecture.

  • Docker Client: The docker command used by the user.
  • Docker Daemon: Background service dockerd that manages containers, images, networks, and volumes.
  • Docker Registry: Stores Docker images. Docker Hub is the default public registry.
  • Docker Objects: Images, containers, networks, volumes, and plugins.
When you run docker run nginx, the Docker client sends the request to the Docker daemon. If the image is missing locally, the daemon pulls it from a registry, creates a container, and starts it.

4. Installing Docker on Various Platforms

Training recommendation: For laptops/desktops, use Docker Desktop. For Linux servers, use Docker Engine. Avoid Docker Toolbox except for legacy historical discussion.

4.1 Installing Docker on Windows

Recommended: Docker Desktop with WSL 2

Docker Desktop is the normal installation path for modern Windows machines.

Prerequisites

  • Windows 10/11 64-bit.
  • Virtualization enabled in BIOS/UEFI.
  • WSL 2 enabled.
  • Administrator access for installation.

Step-by-step

  1. Download Docker Desktop for Windows from the official Docker website.
  2. Run the installer.
  3. Keep Use WSL 2 instead of Hyper-V selected where applicable.
  4. Restart the machine if prompted.
  5. Open Docker Desktop.
  6. Open PowerShell or Windows Terminal and verify:
docker version
docker info
docker run hello-world

Optional: Enable WSL 2 manually

wsl --install
wsl --set-default-version 2
On Windows, Linux containers run inside a lightweight Linux VM through WSL 2. This is normal.

4.2 Installing Docker on macOS

Recommended: Docker Desktop for Mac

Choose the correct installer based on your CPU:

  • Apple Silicon: M1, M2, M3, M4 series.
  • Intel: Older Intel-based Mac machines.

Step-by-step

  1. Download Docker Desktop for Mac from the official Docker website.
  2. Open the downloaded .dmg file.
  3. Drag Docker into the Applications folder.
  4. Open Docker Desktop from Applications.
  5. Allow required permissions when macOS prompts.
  6. Verify using Terminal:
docker version
docker info
docker run hello-world

4.3 Installing Docker on Linux

On Linux, there are two common options:

Option Best For Notes
Docker Engine Servers, cloud VMs, production-like training labs. CLI-first, lightweight, no desktop GUI required.
Docker Desktop for Linux Developer laptops needing GUI features. Includes desktop UI and integrated tools.

Ubuntu / Debian Example: Install Docker Engine from Official Repository

# 1. Remove old conflicting packages if present
sudo apt-get remove -y docker docker-engine docker.io containerd runc || true

# 2. Update package index and install prerequisites
sudo apt-get update
sudo apt-get install -y ca-certificates curl gnupg

# 3. Add Docker official GPG key
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | \
  sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg

sudo chmod a+r /etc/apt/keyrings/docker.gpg

# 4. Add Docker repository
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] \
  https://download.docker.com/linux/ubuntu \
  $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# 5. Install Docker packages
sudo apt-get update
sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

# 6. Verify
sudo docker version
sudo docker run hello-world

Allow non-root user to run Docker

# Add current user to docker group
sudo usermod -aG docker $USER

# Apply group change without full logout/login
newgrp docker

# Verify without sudo
docker run hello-world
Security note: Membership in the docker group is effectively root-equivalent on the host. Add only trusted users.

Start and enable Docker service

sudo systemctl enable --now docker
sudo systemctl status docker

RHEL / Rocky Linux / AlmaLinux Example

# 1. Install yum-utils
sudo dnf install -y yum-utils

# 2. Add Docker repository
sudo dnf config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo

# 3. Install Docker Engine
sudo dnf install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

# 4. Start Docker
sudo systemctl enable --now docker

# 5. Verify
sudo docker run hello-world

Fedora Example

sudo dnf install -y dnf-plugins-core
sudo dnf config-manager --add-repo https://download.docker.com/linux/fedora/docker-ce.repo
sudo dnf install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
sudo systemctl enable --now docker
sudo docker run hello-world

5. Docker Desktop vs Docker Toolbox

Feature Docker Desktop Docker Toolbox
Status Current Docker product for desktop use. Deprecated/retired legacy solution.
Platform Windows, macOS, Linux. Old Windows/macOS systems.
Virtualization Uses WSL 2/Hyper-V on Windows; native virtualization framework on macOS. Used VirtualBox and Docker Machine.
GUI Includes Docker Dashboard. No modern Docker Desktop dashboard.
Compose Modern docker compose plugin. Older separate tooling.
Recommendation Use for modern desktop/laptop setups. Do not use for new training or production work.
Docker Toolbox may appear in old blogs or videos. For current learning and training, use Docker Desktop or Docker Engine.

6. Post-Installation Verification

6.1 Check Docker Client and Server Version

docker version

Expected output includes two sections:

  • Client: Docker CLI installed on your machine.
  • Server: Docker daemon/engine running properly.

6.2 Check Docker System Information

docker info

This shows storage driver, number of containers, images, Docker root directory, plugins, runtime, and operating system details.

6.3 Run Test Container

docker run hello-world

This command verifies that Docker can:

  1. Contact the Docker daemon.
  2. Pull an image from Docker Hub.
  3. Create a container.
  4. Run the container successfully.

7. Basic Docker CLI Commands

7.1 docker version

Shows Docker client and server version details.

docker version

7.2 docker info

Displays detailed information about Docker installation.

docker info

7.3 docker pull

Downloads an image from a registry.

docker pull nginx
docker pull ubuntu:24.04
docker pull redis:7

7.4 docker run

Creates and starts a new container from an image.

# Run a simple container
docker run hello-world

# Run Ubuntu interactively
docker run -it ubuntu:24.04 bash

# Run Nginx in background and expose port 8080
docker run -d --name web1 -p 8080:80 nginx

# Run and automatically remove after exit
docker run --rm alpine echo "Hello from Alpine"

Important options for docker run

OptionMeaningExample
-dRun in detached/background mode.docker run -d nginx
-itInteractive terminal.docker run -it ubuntu bash
--nameAssign a container name.--name web1
-pPublish container port to host.-p 8080:80
-vMount volume or host path.-v mydata:/data
--rmRemove container after it exits.docker run --rm alpine

7.5 docker push

The docker push command uploads a local Docker image to a registry such as Docker Hub. Before pushing, the image must already exist locally. Do not tag or push an image name that is not present in docker images.

Correct workflow: verify, tag, then push

# 1. Check which images exist locally
docker images

# 2. Example: assume this image already exists locally
# REPOSITORY              TAG
# docker-html-demo        1.0

# 3. Login to Docker Hub
docker login

# 4. Tag the existing local image for your Docker Hub account
docker tag docker-html-demo:1.0 your-dockerhub-username/docker-html-demo:1.0

# 5. Verify that the new tagged image exists locally
docker images | grep docker-html-demo

# 6. Push the image to Docker Hub
docker push your-dockerhub-username/docker-html-demo:1.0

If the image does not exist yet, create it first

# Create a simple image before pushing
mkdir docker-push-demo
cd docker-push-demo

cat > index.html <<'EOF'
<h1>Hello from Docker Push Demo</h1>
EOF

cat > Dockerfile <<'EOF'
FROM nginx:latest
COPY index.html /usr/share/nginx/html/index.html
EOF

docker build -t docker-html-demo:1.0 .

# Now verify
docker images | grep docker-html-demo
Replace your-dockerhub-username with your real Docker Hub username. The local image name used in this document is docker-html-demo:1.0, which is created in Example 3.

7.6 docker ps

Lists containers.

# Show running containers
docker ps

# Show all containers, including stopped containers
docker ps -a

7.7 docker images

Lists local Docker images.

docker images

7.8 docker container start and docker container stop

Start or stop an existing container.

docker container stop web1
docker container start web1

7.9 docker container rm

Removes stopped containers.

# Remove one stopped container
docker container rm web1

# Force remove a running container
docker container rm -f web1

# Remove all stopped containers
docker container prune

7.10 docker exec

Runs a command inside an already running container.

# Open shell inside running Nginx container
docker exec -it web1 sh

# Run one command inside a container
docker exec web1 nginx -v

7.11 docker logs

Displays logs from a container.

# Show logs
docker logs web1

# Follow logs live
docker logs -f web1

# Show last 20 lines
docker logs --tail 20 web1

7.12 docker network

Manages Docker networks.

# List networks
docker network ls

# Create a user-defined bridge network
docker network create appnet

# Inspect network details
docker network inspect appnet

# Remove network
docker network rm appnet

7.13 docker volume

Manages persistent Docker volumes.

# List volumes
docker volume ls

# Create volume
docker volume create appdata

# Inspect volume
docker volume inspect appdata

# Remove volume
docker volume rm appdata

8. Complete Hands-on Labs with Verification

Lab design: These labs use simple, repeatable commands. Each lab includes cleanup, execution, verification, and expected output hints so trainees can confirm their work.
If you get permission denied while trying to connect to the Docker API at unix:///var/run/docker.sock, fix Docker permissions first: 
sudo usermod -aG docker $USER
newgrp docker
docker run hello-world
Do not use chmod 666 /var/run/docker.sock.

Lab 1: Verify Docker Installation using hello-world

Objective: Confirm that Docker client, Docker daemon, image pull, container creation, and container execution are working.

Step 1: Check Docker version

docker version

Verify: Output should show both Client and Server sections.

Step 2: Check Docker system information

docker info

Verify: Output should show details such as Containers, Images, Server Version, and Storage Driver.

Step 3: Run the test container

docker run hello-world

Expected result: You should see a message similar to:

Hello from Docker!
This message shows that your installation appears to be working correctly.

Step 4: Check container history

docker ps -a

Verify: You should see a stopped container created from the hello-world image.

Cleanup

docker container prune -f

Verify cleanup:

docker ps -a

Lab 2: Pull and Run an Nginx Web Server

Objective: Download an image, run a web server container, expose it on a host port, verify browser/curl access, and clean up safely.

Step 1: Remove old container if it exists

docker rm -f mynginx 2>/dev/null || true

Step 2: Pull the Nginx image

docker pull nginx:latest

Verify:

docker images nginx

You should see nginx with tag latest.

Step 3: Run Nginx container

docker run -d --name mynginx -p 8080:80 nginx:latest

Verify container is running:

docker ps --filter "name=mynginx"

You should see status similar to Up and port mapping similar to 0.0.0.0:8080->80/tcp.

Step 4: Test from command line

curl -I http://localhost:8080

Expected result:

HTTP/1.1 200 OK
Server: nginx

Step 5: Test from browser

http://localhost:8080

Expected result: Nginx welcome page should open.

Step 6: View logs

docker logs mynginx

Verify: After using browser or curl, access logs should appear.

Step 7: Stop and remove container

docker stop mynginx
docker rm mynginx

Verify cleanup:

docker ps -a --filter "name=mynginx"

No container should be listed.

Lab 3: Run Ubuntu Container Interactively

Objective: Start an interactive Linux container, run commands inside it, exit, restart it, and remove it.

Step 1: Remove old container if it exists

docker rm -f ubuntu-lab 2>/dev/null || true

Step 2: Start Ubuntu container interactively

docker run -it --name ubuntu-lab ubuntu:24.04 bash

You are now inside the container shell. The prompt may look like root@container-id:/#.

Step 3: Run commands inside container

cat /etc/os-release
whoami
hostname
exit

Expected result: whoami should show root. The OS release should show Ubuntu 24.04 details.

Step 4: Verify container exists but is stopped

docker ps -a --filter "name=ubuntu-lab"

Verify: Status should show Exited.

Step 5: Start it again and execute a command

docker start ubuntu-lab
docker exec ubuntu-lab cat /etc/os-release

Verify: The command should print Ubuntu OS details without opening an interactive shell.

Step 6: Open shell again

docker exec -it ubuntu-lab bash

Exit again:

exit

Cleanup

docker rm -f ubuntu-lab

Verify cleanup:

docker ps -a --filter "name=ubuntu-lab"

Lab 4: Create a Simple Custom HTML Image using Dockerfile

Objective: Build a custom Docker image using a Dockerfile and serve a simple HTML page through Nginx.

Step 1: Create clean project directory

rm -rf docker-html-demo
mkdir docker-html-demo
cd docker-html-demo

Step 2: Create index.html

cat > index.html <<'EOF'
<!DOCTYPE html>
<html>
<head>
    <title>Docker Demo</title>
</head>
<body>
    <h1>Hello from Docker!</h1>
    <p>This page is served from a custom Docker image.</p>
</body>
</html>
EOF

Verify file:

cat index.html

Step 3: Create Dockerfile

cat > Dockerfile <<'EOF'
FROM nginx:latest
COPY index.html /usr/share/nginx/html/index.html
EOF

Verify Dockerfile:

cat Dockerfile

Step 4: Build the image

docker build -t docker-html-demo:1.0 .

Verify image exists:

docker images | grep docker-html-demo

You should see docker-html-demo with tag 1.0.

Step 5: Run the custom image

docker rm -f html-demo 2>/dev/null || true
docker run -d --name html-demo -p 8081:80 docker-html-demo:1.0

Verify container:

docker ps --filter "name=html-demo"

Step 6: Test the web page

curl http://localhost:8081

Expected result: Output should include:

Hello from Docker!
This page is served from a custom Docker image.

Step 7: Check logs

docker logs html-demo

Cleanup

docker rm -f html-demo
cd ..
rm -rf docker-html-demo

Optional image cleanup:

docker image rm docker-html-demo:1.0

Lab 5: Push a Custom Image to Docker Hub

Objective: Tag and push a local custom image to Docker Hub.

This lab requires a Docker Hub account and internet access. Replace your-dockerhub-username with your actual Docker Hub username.

Step 1: Make sure the image exists locally

docker images | grep docker-html-demo

If nothing is shown: Re-run Lab 4, but do not remove the image at the end.

Step 2: Login to Docker Hub

docker login

Verify: Login should complete successfully.

Step 3: Tag the local image

docker tag docker-html-demo:1.0 your-dockerhub-username/docker-html-demo:1.0

Verify tag:

docker images | grep docker-html-demo

You should see both:

docker-html-demo                              1.0
your-dockerhub-username/docker-html-demo      1.0

Step 4: Push the image

docker push your-dockerhub-username/docker-html-demo:1.0

Verify: Docker should upload layers and finish without error.

Step 5: Pull and test the pushed image

docker rm -f pushed-demo 2>/dev/null || true
docker run -d --name pushed-demo -p 8082:80 your-dockerhub-username/docker-html-demo:1.0
curl http://localhost:8082

Expected result: The same custom HTML page should be returned.

Cleanup

docker rm -f pushed-demo

Lab 6: Practice Container Lifecycle Commands

Objective: Practice run, ps, logs, stop, start, exec, and rm.

Step 1: Run a lightweight long-running container

docker rm -f lifecycle-lab 2>/dev/null || true
docker run -d --name lifecycle-lab alpine:latest sh -c "while true; do echo Docker lifecycle lab; sleep 5; done"

Step 2: Verify it is running

docker ps --filter "name=lifecycle-lab"

Step 3: Check logs

docker logs lifecycle-lab

Step 4: Execute a command inside it

docker exec lifecycle-lab cat /etc/os-release

Step 5: Stop the container

docker stop lifecycle-lab

Verify stopped state:

docker ps -a --filter "name=lifecycle-lab"

Step 6: Start it again

docker start lifecycle-lab
docker ps --filter "name=lifecycle-lab"

Cleanup

docker rm -f lifecycle-lab

9. Docker Network and Volume Examples

9.1 Docker Network Concept

Docker networks allow containers to communicate with each other. The default bridge network works, but user-defined bridge networks are better because they provide automatic DNS-based name resolution between containers.

9.2 Create Network and Run Containers

# Create network
docker network create appnet

# Run Nginx on appnet
docker run -d --name web --network appnet nginx

# Run Alpine and test DNS resolution
docker run --rm -it --network appnet alpine sh

# Inside Alpine container:
ping web
exit

9.3 Docker Volume Concept

Containers are temporary by design. If a container is deleted, data written inside the container can be lost. Docker volumes provide persistent storage managed by Docker.

9.4 Create and Use a Volume

# Create volume
docker volume create webdata

# Run Nginx with volume mounted
docker run -d --name vol-nginx -p 8083:80 \
  -v webdata:/usr/share/nginx/html \
  nginx

# Create a file inside the container volume
docker exec -it vol-nginx sh -c 'echo "<h1>Data from Docker Volume</h1>" > /usr/share/nginx/html/index.html'

# Test
curl http://localhost:8083

# Remove container
docker rm -f vol-nginx

# Run another container using the same volume
docker run -d --name vol-nginx2 -p 8084:80 \
  -v webdata:/usr/share/nginx/html \
  nginx

# Data still exists
curl http://localhost:8084

9.5 Cleanup Network and Volume

docker rm -f web vol-nginx2 2>/dev/null || true
docker network rm appnet
docker volume rm webdata

10. Common Troubleshooting

Problem Likely Cause Solution
Cannot connect to the Docker daemon Docker service not running or user lacks permission. Run sudo systemctl status docker. Start with sudo systemctl start docker. On Linux, check docker group membership.
permission denied while trying to connect to Docker daemon socket User not in docker group. sudo usermod -aG docker $USER, then logout/login or run newgrp docker.
Port already allocated Another process/container is using the host port. Use another host port, e.g. -p 8081:80, or stop the conflicting process.
Container exits immediately Main process completed or failed. Check logs using docker logs container-name.
Image pull is slow/failing Network, proxy, DNS, or registry issue. Check internet, proxy settings, Docker Desktop settings, or registry login.
Docker Desktop not starting on Windows WSL 2, virtualization, or Hyper-V issue. Enable virtualization in BIOS/UEFI, update WSL, restart Docker Desktop.

Useful Diagnostic Commands

docker version
docker info
docker ps -a
docker images
docker logs <container-name>
docker inspect <container-name>
docker system df
docker system prune
Be careful with docker system prune. It removes unused containers, networks, images, and build cache.
